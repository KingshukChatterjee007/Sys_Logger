### SYS_LOGGER DEPLOYMENT SUMMARY - MARCH 8, 2026 ###

0. POSTGRESQL INSTALLATION GUIDE
================================
A. WINDOWS:
   1. Download the installer from: https://www.enterprisedb.com/downloads/postgres-postgresql-downloads
   2. Run the .exe and follow the wizard (Install Server, pgAdmin, and Command Line Tools).
   3. Set a password for the 'postgres' user (Remember this for your .env).
   4. Open 'pgAdmin 4' and create a new database named 'sys_logger'.

B. LINUX (UBUNTU/DEBIAN):
   sudo apt update && sudo apt install postgresql postgresql-contrib
   sudo -u postgres psql -c "CREATE DATABASE sys_logger;"

C. MACOS:
   brew install postgresql@14
   brew services start postgresql@14
   createdb sys_logger

1. OVERVIEW OF CHANGES
======================
The project has shifted from a "Local JSON + Global Broadcast" model to a "PostgreSQL-Backed + Multi-Tenant Isolated" architecture. This prevents Organization Admins (like TEST or NIELIT) from seeing each other's data.

2. BACKEND (sys_logger.py)
==========================
- DEPRECATED: 'units_db.json' is no longer the Source of Truth.
- NEW: PostgreSQL is now required.
- TABLES:
    - 'systems': Stores unit metadata. Enforces 'org_id' foreign key.
    - 'system_metrics': Stores all telemetry (CPU, GPU, RAM, etc).
- SECURITY:
    - Added JWT Authentication.
    - Added BCrypt password hashing.
    - Enforced SQL-level filtering (WHERE org_id = %current_user_org_id).

3. DATABASE SCHEMA
==================
Ensure the following tables exist on the baremetal server:
- 'users': user_id, email, password_hash, role, org_id
- 'organizations': org_id, name, created_at
- 'systems': system_id, system_name, org_id (FK), status, last_seen, etc.
- 'system_metrics': id, system_id (FK), cpu, ram, gpu, timestamp...

4. FRONTEND CONFIGURATION
=========================
- .env.local: MUST have NEXT_PUBLIC_API_URL=http://localhost:5010 (or the server public IP).
- Authorization: 'src/app/api/proxyUtils.ts' now passes the Bearer token to the backend.
- Proxy: 'src/app/api/auth/login/route.ts' explicitly forwards login requests to resolve 404s.
